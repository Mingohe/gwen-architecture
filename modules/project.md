## 1. 模块概述

### 1.1 模块定位

项目管理模块是GWEN系统的项目组织模块，支持项目创建、成员管理、Issue管理和第三方平台集成，用于组织和管理项目相关的任务和问题。

### 1.2 核心业务功能

- **项目创建**: 创建和管理项目，设置项目基本信息
- **成员管理**: 添加和管理项目成员
- **Issue管理**: 创建、跟踪和关闭项目Issue
- **第三方平台集成**: 集成GitHub、GitLab、Jira等第三方平台
- **项目财务关联**: 关联项目合同和收支记录

### 1.3 业务价值

- **项目组织**: 通过项目组织和管理相关任务和Issue
- **问题跟踪**: 通过Issue跟踪项目中的问题
- **平台集成**: 通过第三方平台集成实现统一管理
- **财务关联**: 通过项目财务关联实现项目成本管理

## 2. 核心业务概念

### 2.1 项目状态

- **正常** (0): 项目正常运行
- **待绑定第三方平台** (-1): 项目创建但未绑定第三方平台

### 2.2 Issue状态

| 状态 | 状态值 | 说明 |
|------|--------|------|
| 待处理 | 0 | Issue已创建，等待处理 |
| 处理中 | 1 | Issue正在处理中 |
| 已完成 | 2 | Issue已完成 |
| 已关闭 | 3 | Issue已关闭 |

### 2.3 Issue优先级

- **低** (0): 低优先级
- **中** (1): 中优先级
- **高** (2): 高优先级

### 2.4 第三方平台

支持的第三方平台：

- **GitHub**: 代码仓库和Issue管理
- **GitLab**: 代码仓库和Issue管理
- **Jira**: 项目管理和Issue跟踪

## 3. 业务流程

### 3.1 项目创建流程

```
1. 用户创建项目
   ↓
2. 填写项目信息
   - 项目名称
   - 项目描述
   - 项目类型
   - 负责人
   - 货币类型
   - 底薪
   ↓
3. 选择第三方平台（可选）
   ↓
4. 创建项目
   ↓
5. 添加项目成员
   ↓
6. 项目创建完成
```

### 3.2 Issue创建流程

```
1. 用户在项目中创建Issue
   ↓
2. 填写Issue信息
   - 标题
   - 描述
   - 优先级
   - 关联任务（可选）
   ↓
3. 创建Issue
   ↓
4. 如果关联了第三方平台，同步到第三方平台
   ↓
5. Issue出现在项目Issue列表中
```

### 3.3 Issue处理流程

```
待处理 (0)
  ↓ [开始处理]
处理中 (1)
  ↓ [完成]
已完成 (2)
  ↓ [关闭]
已关闭 (3)
```

### 3.4 第三方平台集成流程

```
1. 项目创建时选择第三方平台
   ↓
2. 配置第三方平台认证信息
   ↓
3. 绑定第三方平台项目
   ↓
4. 同步Issue到第三方平台
   ↓
5. 第三方平台Issue变更同步回系统
```

## 4. 业务规则

### 4.1 项目创建规则

1. **名称必填**: 项目名称不能为空
2. **负责人必填**: 项目必须指定负责人
3. **成员限制**: 项目可以有最大参与者数限制
4. **财务设置**: 项目可以设置货币类型和底薪

### 4.2 项目成员规则

1. **添加权限**: 只有项目负责人可以添加成员
2. **移除权限**: 只有项目负责人可以移除成员
3. **成员角色**: 项目成员可以参与项目任务和Issue

### 4.3 Issue管理规则

1. **创建权限**: 项目成员可以创建Issue
2. **关联任务**: Issue可以关联到任务，跟踪任务相关问题
3. **状态流转**: Issue状态必须按照预定义流程流转
4. **关闭权限**: 只有Issue创建者或项目负责人可以关闭Issue

### 4.4 第三方平台集成规则

1. **认证要求**: 集成第三方平台需要配置认证信息
2. **同步机制**: Issue创建和更新时同步到第三方平台
3. **双向同步**: 第三方平台Issue变更同步回系统
4. **状态映射**: 第三方平台状态映射到系统状态

### 4.5 项目财务关联规则

1. **合同关联**: 项目可以关联多个合同
2. **收支记录**: 项目相关的收支记录关联到项目
3. **财务统计**: 项目可以统计收支情况

## 5. 模块关联

### 5.1 与任务系统关联

- Issue可以关联到任务，跟踪任务相关问题
- 任务可以关联到项目，组织项目任务
- 项目成员可以创建和管理任务

### 5.2 与财务系统关联

- 项目可以关联合同
- 项目相关的收支记录关联到项目
- 项目可以统计财务情况

### 5.3 与用户系统关联

- 项目负责人和成员都是系统中的用户
- 项目权限基于用户角色

### 5.4 与代码文档系统关联

- 项目可以关联代码仓库
- Issue可以关联代码文档
