## 1. 模块概述

### 1.1 模块定位

会议系统模块是GWEN系统的协作沟通模块，支持会议预约、实时协作、会议笔记和会议摘要功能，主要用于任务验收和团队讨论。

### 1.2 核心业务功能

- **会议预约**: 创建和管理会议，设置会议时间和参与者
- **参与者管理**: 邀请参与者，管理参与者确认状态
- **会议笔记**: 实时协作编辑会议笔记
- **会议摘要**: 生成和确认会议摘要
- **实时协作**: 通过WebSocket实现会议实时通信

### 1.3 业务价值

- **任务验收**: 通过会议进行任务验收，确保任务质量
- **团队沟通**: 支持团队实时沟通和讨论
- **记录沉淀**: 通过会议笔记和摘要记录会议内容
- **流程规范**: 通过会议流程规范任务验收过程

## 2. 核心业务概念

### 2.1 会议状态

| 状态 | 状态值 | 说明 |
|------|--------|------|
| OK | 0 | 普通状态（已预约） |
| CANCELED | 1 | 取消/删除 |
| ON_GOING | 2 | 正在开会 |
| FINISHED | 3 | 会议完成 |
| CONFIRMING | 4 | 会议正在确认摘要 |

### 2.2 参与者角色

- **主持人**: 会议创建者，可以管理会议和参与者
- **参与者**: 被邀请参加会议的用户

### 2.3 参与者状态

- **待确认** (0): 已邀请但未确认
- **已确认** (1): 已确认参加
- **已拒绝** (2): 拒绝参加

### 2.4 会议类型

- **任务验收会议**: 关联任务，用于任务验收
- **普通会议**: 不关联任务，用于团队讨论

## 3. 业务流程

### 3.1 会议预约流程

```
1. 用户选择任务（可选）
   ↓
2. 点击"预约会议"
   ↓
3. 填写会议信息
   - 会议主题
   - 开始时间
   - 结束时间
   - 参与者列表
   ↓
4. 创建会议
   ↓
5. 发送会议邀请给参与者
   ↓
6. 参与者收到通知，确认/拒绝参加
```

### 3.2 会议进行流程

```
1. 会议开始时间到达
   ↓
2. 会议状态变为 ON_GOING
   ↓
3. 参与者加入会议
   ↓
4. 实时协作编辑会议笔记
   ↓
5. 讨论任务验收（如果关联任务）
   ↓
6. 会议结束
   ↓
7. 会议状态变为 FINISHED
   ↓
8. 生成会议摘要
   ↓
9. 确认摘要（CONFIRMING）
   ↓
10. 摘要确认完成
```

### 3.3 任务验收流程

```
1. 任务提交（STATUS_SUBMITTED）
   ↓
2. 创建任务验收会议
   ↓
3. 会议中进行验收讨论
   ↓
4. 验收结果：
   ├─ [通过] → 任务状态变为 CHECKED
   └─ [不通过] → 任务状态变为 STRIKED
   ↓
5. 会议后确认任务完成状态
```

### 3.4 会议笔记流程

```
1. 会议进行中
   ↓
2. 参与者实时编辑会议笔记
   ↓
3. 笔记内容通过WebSocket实时同步
   ↓
4. 会议结束后
   ↓
5. 笔记保存为会议记录
   ↓
6. 可以关联到任务或其他笔记
```

## 4. 业务规则

### 4.1 会议创建规则

1. **主题必填**: 会议主题不能为空
2. **时间验证**: 结束时间必须晚于开始时间
3. **参与者限制**: 至少需要一个参与者
4. **任务关联**: 会议可以关联任务，用于任务验收

### 4.2 会议状态流转规则

1. **创建**: 会议创建时状态为 OK
2. **开始**: 会议开始时间到达时状态变为 ON_GOING
3. **结束**: 会议结束时间到达时状态变为 FINISHED
4. **确认摘要**: 会议结束后进入 CONFIRMING 状态
5. **取消**: 会议可以随时取消，状态变为 CANCELED

### 4.3 参与者管理规则

1. **邀请权限**: 只有会议创建者可以邀请参与者
2. **确认机制**: 参与者需要确认是否参加
3. **角色分配**: 会议创建者默认为主持人
4. **通知机制**: 参与者收到会议邀请和状态变更通知

### 4.4 会议笔记规则

1. **编辑权限**: 所有参与者都可以编辑会议笔记
2. **实时同步**: 笔记内容通过WebSocket实时同步
3. **关联任务**: 笔记可以关联到任务，记录任务相关讨论
4. **保存机制**: 会议结束后笔记自动保存

### 4.5 会议摘要规则

1. **生成时机**: 会议结束后生成摘要
2. **确认流程**: 需要参与者确认摘要内容
3. **修改权限**: 只有会议创建者可以修改摘要
4. **关联任务**: 摘要可以关联到任务，用于任务验收记录

### 4.6 任务验收规则

1. **验收时机**: 任务提交后通过会议进行验收
2. **验收结果**: 会议中决定任务是否通过验收
3. **状态更新**: 验收结果直接影响任务状态
4. **记录保存**: 验收过程和结果记录在会议笔记和摘要中

---

## 5. 模块关联

### 5.1 与任务系统关联

- 会议可以关联任务，用于任务验收
- 任务验收结果通过会议决定
- 会议笔记和摘要可以关联到任务

### 5.2 与笔记系统关联

- 会议笔记可以关联到个人笔记
- 会议讨论内容可以记录到笔记中

### 5.3 与用户系统关联

- 会议创建者和参与者都是系统中的用户
- 会议通知涉及用户通知系统
